# TaDV Demo Frontend

Interactive playground interface for exploring data quality constraints generated by TaDV.

## 🔒 Reproducible Environment Setup

### Prerequisites

**Required:**
- Node.js `>= 20.11.0` (LTS recommended)
- npm `>= 10.0.0`

**Recommended:** Use [nvm](https://github.com/nvm-sh/nvm) (Node Version Manager) for version management:

```bash
# Install nvm (if not already installed)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Use the pinned Node version (reads from .nvmrc)
nvm install
nvm use

# Verify versions
node --version  # Should be >= 20.11.0
npm --version   # Should be >= 10.0.0
```

### Installation

```bash
# Install exact dependencies from package-lock.json
npm ci

# Or if package-lock.json doesn't exist yet
npm install
```

**Why `npm ci`?**
- Uses exact versions from `package-lock.json` (like `poetry install` or `uv sync`)
- Faster than `npm install` in CI environments
- Fails if package.json and package-lock.json are out of sync
- Always produces identical node_modules

### Development

```bash
# Start dev server (http://localhost:3000)
npm run dev

# Type check
npm run build

# Lint code
npm run lint
```

## 📦 Reproducibility Strategy

### 1. Version Pinning
- **Node.js version:** Pinned in `.nvmrc` (similar to `.python-version`)
- **Package versions:** Locked in `package-lock.json` (similar to `poetry.lock` or `uv.lock`)
- **Engine requirements:** Enforced in `package.json` engines field

### 2. Lock Files
```bash
# package-lock.json is committed to git
# Always use 'npm ci' in CI/production
# Use 'npm install' only when adding/updating packages
```

### 3. Dependency Management

**Adding new packages:**
```bash
npm install <package>          # Adds to dependencies
npm install -D <package>        # Adds to devDependencies
git add package.json package-lock.json
```

**Updating packages:**
```bash
npm update <package>            # Update specific package
npm outdated                    # Check for outdated packages
```

### 4. Alternative: pnpm (Optional, More Like uv/Poetry)

If you want even better reproducibility (faster, disk-efficient, stricter):

```bash
# Install pnpm globally
npm install -g pnpm

# Use pnpm instead of npm
pnpm install                    # Install dependencies
pnpm add <package>              # Add dependency
pnpm run dev                    # Run scripts

# Creates pnpm-lock.yaml (commit this)
```

**pnpm benefits:**
- Content-addressable storage (saves disk space)
- Stricter dependency resolution
- Faster installs
- Better monorepo support
